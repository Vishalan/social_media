# CommonCreed AI Avatar Content Pipeline

## Problem Frame

@commoncreed is an established Instagram channel with ~8,000 followers built around the owner's real face and personal brand. The goal is to automate daily AI avatar-driven tech news content — 2-3 vertical shorts per day — to grow that audience and generate income through affiliate marketing. This deliberately uses the owner's real face (not a faceless persona) to leverage the existing brand and community trust. The general `social_media` project uses a faceless strategy for new channels; @commoncreed is a separate workstream for this established personal brand.

The avatar must achieve ~80% of HeyGen quality using open-source models self-hosted on cloud GPU (RunPod/Vast.ai). The pipeline integrates into the existing `social_media` project and requires only a daily approval step from the owner (under 30 minutes/day).

**Revenue model:** Every video description includes 2-3 affiliate links relevant to the news topic (AI tools, tech products, courses). This scales passively with views and follower growth.

## Validation Gate (Required Before Full Build)

Before building R4-R6 (avatar generation, video editing, and the VideoEditor module), a quality spike must be completed:

- **VS1.** Run EchoMimic V3 on the owner's reference footage on a target GPU instance (RunPod/Vast.ai RTX 4090). Score output on lip-sync accuracy, unnatural motion, and identity consistency on a 1-5 scale. Target: ≥4/5 overall.
- **VS2.** Measure inference time for a 45-second avatar clip. Verify GPU cost fits within the $2/video cap at the target instance rate.
- **VS3.** If EchoMimic V3 fails (score <4/5 or cost exceeds cap), evaluate Duix.Avatar as the fallback. If both fail, HeyGen API ($29-89/month) is the bridge until a better open-source option matures.

## Requirements

- R1. The pipeline sources 2-3 tech news topics per day from RSS/API feeds (Google News RSS, Hacker News, NewsData.io). Raw feed content is sanitized (HTML stripped, field lengths capped) before being passed to any LLM to prevent prompt injection. Topics are deduplicated against the `AnalyticsTracker` SQLite history (skip any topic whose URL or normalized title appeared in the past 7 days). If fewer than 2 suitable topics are found, the pipeline sends a Telegram alert and skips generation for that day rather than posting low-quality content.
- R2. For each topic, an AI script is generated (45-90 seconds spoken, ~120-200 words) using the existing `ScriptGenerator` class.
- R3. Voiceover is synthesized using ElevenLabs via the existing `VoiceGenerator` class.
- R4. An AI avatar video (A-roll) is generated from the owner's reference footage using **EchoMimic V3** (pending VS1 validation), producing a portrait/upper-body presenter clip synchronized to the voiceover audio. If avatar generation produces a degraded or unusable clip (detected by face-presence check or quality score below threshold), the pipeline retries once with a different seed before falling back to a b-roll-only format and flagging the owner via Telegram.
- R5. B-roll footage (supporting visuals) is generated via existing ComfyUI workflows (CogVideoX-5B or Wan2.1) prompted with topic-relevant descriptions. Avatar generation (R4) and B-roll generation (R5) run **sequentially on a single RTX 4090 instance** with model unload between steps — concurrent use is not feasible within 24GB VRAM.
- R6. A final vertical short (9:16, 1080x1920) is assembled by a new `VideoEditor` module (in `scripts/video_edit/`) using MoviePy and FFmpeg with the following production rules:
  - **Full-screen layout**: Avatar fills the frame (for hooks and CTA segments).
  - **Half-screen layout**: Avatar occupies the bottom half; B-roll plays in the top half (for explanatory segments).
  - **Layout switching logic**: TBD in planning — options include fixed-ratio (e.g., first 5 seconds full-screen, middle body half-screen, last 5 seconds full-screen) or script segment markers generated by `ScriptGenerator`. Planning must resolve this before VideoEditor can be implemented.
  - **Silence trimming**: Dead air removed using whisper word-level timestamps already available from the caption step (avoids the auto-editor dependency conflict).
  - **Aesthetic captions**: Word-level animated subtitles burned in via faster-whisper + FFmpeg, styled (bold, outlined, centered). Caption style parameters (font, size, color, animation) are deferred to planning.
- R7. The assembled video is uploaded to a pre-signed URL (short TTL, non-public) and sent via **Telegram bot** to the owner with a preview link and Approve/Reject buttons. The bot must validate that all incoming callbacks originate from the owner's Telegram user ID (hardcoded allowlist). All other user IDs receive no response. Bot token is stored in `.env`, never in source or logs. **Rejection handling**: if the owner rejects a video, the pipeline generates one retry on a new seed; if that is also rejected, the topic is skipped and logged. If the owner does not respond within 4 hours, the video is auto-rejected and logged for review.
- R8. Upon owner approval, the video file is posted to Instagram Reels (primary), TikTok, and YouTube Shorts via the existing `SocialPoster` class. Each post includes: (a) 2-3 affiliate links in the description relevant to the video topic, (b) AI-generated content disclosure label as required by each platform's synthetic media policy (Instagram AI label, TikTok synthetic media tag, YouTube altered/synthetic content disclosure).
- R9. All new scripts (avatar generation, video editing, news sourcing) live within the existing `scripts/` package structure.
- R10. All new ComfyUI workflows for avatar generation are stored in `comfyui_workflows/`.
- R11. New GPU deployment scripts for EchoMimic V3 are added to `deploy/runpod/` and `deploy/vastai/`.

## Success Criteria

- 2-3 approved vertical shorts are posted daily to Instagram Reels, TikTok, and YouTube Shorts.
- **Pre-launch quality gate** (measurable, not audience-dependent): EchoMimic V3 output on the owner's reference footage scores ≥4/5 on blind review for lip-sync accuracy, natural motion, and identity consistency before the pipeline goes live.
- Total owner time: under 30 minutes/day (review + approval only).
- GPU cost per video: under $2 — validated by VS2 inference benchmark before build begins.
- End-to-end pipeline time from news fetch to ready-to-review video: under 2 hours (sequential: avatar generation + b-roll + assembly per video; 3 videos must fit within this window).
- Pipeline fits within the existing `social_media/` project structure with no architectural conflicts.

## Scope Boundaries

- **Not in scope:** Training or fine-tuning avatar models from scratch (use pre-trained models with owner's reference video as conditioning input).
- **Not in scope:** Long-form YouTube video generation (separate from the existing general pipeline).
- **Not in scope:** Building a custom web review UI (Telegram bot is sufficient).
- **Not in scope:** Managing multiple brand identities or channels (only @commoncreed).
- **Not in scope:** Real-time or live-stream avatar generation.
- **Not in scope:** Facebook or LinkedIn in this pipeline iteration.
- **Not in scope (initially):** Duix.Avatar integration — retained as a named fallback if EchoMimic V3 fails the VS1 quality gate.

## Key Decisions

- **EchoMimic V3 as primary avatar model** (pending VS1 validation): Apache 2.0, 12-16GB VRAM (fits RTX 4090), upper-body + portrait, AAAI 2026 paper quality, built on Wan2.1 (already in project). Duix.Avatar is explicitly retained as a fallback — it is a more direct HeyGen clone and should be evaluated if EchoMimic V3 underperforms, not dismissed solely on setup friction. Decision sequence: EchoMimic V3 → Duix.Avatar → HeyGen API bridge.
- **Wan2.2 S2V as a named fallback only**: Not validated during planning unless EchoMimic V3 fails VS1. Has no separate deliverable.
- **Telegram bot for approval**: Net-new infrastructure — not an extension of the existing n8n Slack approval flow. Justification: Telegram allows inline video preview and native button callbacks on mobile, which the n8n Telegram node does not support as cleanly. Bot token stored in `.env`; sender-ID validation required (see R7).
- **Video hosting between assembly and posting**: Assembled videos must be uploaded to a pre-signed/temporary URL before Telegram preview (R7) and Ayrshare/Instagram posting (R8), since Ayrshare requires a publicly accessible video URL, not a local file path. S3, Cloudflare R2, or Ayrshare's own media upload endpoint are candidate solutions — deferred to planning.
- **News sourcing via feedparser + Google News RSS** (primary): Zero cost, no API key. Upgrade to NewsData.io (200 req/day free) if topic filtering quality is insufficient. `ScriptGenerator.suggest_topics()` is used for fallback topic ideas only; primary sourcing is RSS-driven to ensure real-time tech news.
- **Cross-platform posting (Instagram + TikTok + YouTube Shorts)**: Triples reach at no extra production cost; `SocialPoster` already supports all three. AI disclosure labels are required on all platforms.

## Dependencies / Assumptions

- Owner provides a clean reference video of their face for avatar conditioning: **minimum specifications** — 1080p or higher, 24fps+, well-lit (no harsh shadows), neutral or blurred background, speaking naturally with clear lip movement, 10-30 seconds duration, recorded in H.264 or ProRes. The reference footage must not be stored persistently on cloud GPU instances: upload at job time via authenticated API, use ephemeral instance storage only, delete after conditioning is complete.
- RunPod or Vast.ai instance with RTX 4090 (24GB VRAM) is available at ~$0.44-0.69/hr.
- ElevenLabs API key is configured (voice already selected or cloned). **Note:** CLAUDE.md marks this as incomplete — pipeline cannot be tested end-to-end until resolved.
- Instagram, TikTok, and YouTube accounts exist. **Instagram Developer App with advanced access for the Reels API must be submitted immediately in parallel with pipeline development** — approval can take weeks and is a blocking dependency for R8.
- All credentials (ElevenLabs, Ayrshare, YouTube OAuth2, RunPod/Vast.ai, Telegram bot token) are stored in `.env` files only, never committed to source control, delivered to GPU instances via environment injection at runtime, and have separate dev/prod key scopes.
